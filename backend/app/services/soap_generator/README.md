# SOAP Generator

This module generates structured SOAP notes from emergency triage dialogue using an LLM-first pipeline: `generate_soap(payload)` validates a `SOAPRequest`, resolves the configured OpenAI-compatible model endpoint from `config.yaml` and `LLM_API_KEY`, injects scenario metadata and dialogue into a strict clinical prompt, calls the model at low temperature, extracts JSON, normalizes missing or unstable fields into the expected SOAP schema, and validates the final result with Pydantic before returning `scenario_number`, `summary_header`, `soap`, and `brief_summary`. In this Docker project, start the stack from the repository root with `docker compose up --build -d`, then execute SOAP utilities inside the backend service, where the backend code is mounted at `/app`. For a quick generation smoke test, run `docker compose exec -T backend python app/services/soap_generator/soap_test.py --index 0`; to run all scenarios, use `docker compose exec -T backend python app/services/soap_generator/soap_test.py --all`; to include the lightweight ROUGE-L smoke evaluation, add `--eval`. The formal benchmark is under `benchmark/` and evaluates generated SOAP outputs against heuristic gold annotations by flattening gold and predicted SOAP notes into facts, matching them with hybrid TF-IDF character n-gram plus RapidFuzz similarity, and reporting structure validity, must-fact recall, supported-fact precision, section placement, clinical adequacy, safety score, and optional ETEK handbook alignment. To run the sample benchmark inside Docker, use `docker compose exec -T backend sh -lc "cd /app/app/services/soap_generator/benchmark && PYTHONPATH=. python scripts/evaluate_generated_json.py --gold data/gold_annotations.json --pred data/generated_outputs_from_samples.json --handbook_dir data/handbook_index --out_json data/evaluation_results_generated_samples.json"`. The main benchmark score combines required clinical fact coverage, unsupported-content control, correct SOAP section placement, clinical completeness, and safety checks such as negation flips, missing numeric details, and missing urgency language for ATS 1-2 cases.
